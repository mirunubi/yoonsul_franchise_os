# 3 Purpose Of The Phase 3 Spine

< Purpose >

Phase 3 exists to define the runtime universe before subsystem split.
It gives the program a common constitution so future subsystem documents can be written without drifting into ad hoc implementation language.

This spine is meant to:

- define the onboarding and navigation rules for the civilization namespace.
- keep the lane topology easy to find for humans and machine-assisted readers.
- preserve recovery-first and human-override principles in the constitutional layer.
- reserve future bands for subsystem-specific constitutional documents.
- keep implementation out of the onboarding layer.
