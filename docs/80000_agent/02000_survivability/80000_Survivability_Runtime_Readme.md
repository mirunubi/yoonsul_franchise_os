Survivability is the continuity district for Agent Runtime.
It is for keeping agent support useful when connectivity, freshness, or quality degrades.

Growth pressure: high, because survivability tends to expand across offline, degraded, and island-mode scenarios.
Semantic boundary: preserve continuity, but do not overtake recovery or store authority.
Reserved insertion: leave room for future resilience notes without folding them into governance.
