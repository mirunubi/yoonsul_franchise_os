# Purpose

< Purpose >

Correlation Runtime exists to discover relationships.
Correlation Runtime does not determine truth.
Correlation Runtime does not determine causality.
Correlation Runtime does not approve actions.
Correlation Runtime does not execute actions.

This constitution defines the bounded role of correlation inside the operational runtime civilization.

# Scope

< Scope >

This constitution applies to event association, pattern detection, co-occurrence measurement, relationship identification, investigation support, and confidence reporting.

It does not define implementation mechanics.
It does not define SQL, Flutter, RPC, or migration behavior.
It does not replace the observation constitution, the agent constitution, or the human authority constitution.

# 1 Correlation Philosophy

Correlation philosophy is relationship-seeking without truth overreach.

Correlation exists to help humans and runtimes see that separate observations may be related.
It does not claim that related observations have already become explanation, causality, or truth.

Correlation must remain:

- bounded
- explainable
- reviewable
- uncertainty-aware
- human-readable

The point of correlation is to help discovery, not to declare the final answer.

# 2 Constitutional Principles

< Constitutional Principles >

## 2.1 correlation != causation

Things may move together without one causing the other.

Correlation Runtime may point out co-movement, repetition, or alignment, but it must not claim causality as fact.

## 2.2 relationship != explanation

A relationship between events is not the same as an explanation for why those events occurred.

Correlation may reveal a useful connection, but explanation requires further analysis and judgment.

## 2.3 pattern != truth

The presence of a pattern does not make the pattern true in every case.

Patterns may be real, but they are still bounded observations, not universal truth by default.

## 2.4 frequency != significance

Something happening often does not automatically make it important.

Correlation must not confuse repetition with constitutional importance unless that significance is reviewed and justified.

## 2.5 confidence != certainty

Confidence is not certainty.

Correlation may produce confidence scores or confidence-like judgments, but those must remain bounded, visible, and discardable when the context changes.

# 3 Correlation Responsibilities

Correlation Runtime may:

- associate events
- detect patterns
- measure co-occurrence
- identify relationships
- suggest investigation
- produce confidence scores

These responsibilities exist to make the runtime more understandable, not more authoritative.

Correlation should help humans ask better questions.
It should not pretend to be the answer.

# 4 Correlation Boundaries

Correlation Runtime may not:

- declare causality
- approve decisions
- execute actions
- replace human analysis
- rewrite truth
- override runtime ownership

Correlation boundaries are constitutional boundaries.

The runtime may identify that two things are related.
It may not force a conclusion that has not been earned.

# 5 Human Authority Relationship

Human judgment remains final.

Correlation supports human judgment by making relationships visible, but it does not replace analysis or approval.

When correlation is uncertain, the uncertainty must remain visible.
When correlation is stale, the stale interpretation must not be presented as certainty.

Humans may use correlation to investigate.
Humans may not be displaced by correlation output.

# 6 Runtime Federation Relationship

Correlation Runtime participates in runtime federation as a relationship layer, not as a sovereignty layer.

Its federation role is to:

- relate signals across boundaries
- surface cross-runtime patterns
- support investigation across runtimes
- retain local truth boundaries

It must not:

- collapse runtime independence
- turn relationship into control
- replace local truth with federation inference
- act as global authority over meaning

Federation can carry correlation.
It cannot grant causality by force.

# 7 Constitutional Violations

The following are constitutional violations:

- treating correlation as causation
- treating relationship as explanation
- treating pattern as truth
- treating frequency as significance without review
- treating confidence as certainty
- replacing human analysis with correlation output
- using correlation to rewrite truth or ownership boundaries

These are governance failures, not acceptable convenience.

# 8 Runtime Application Scope

This constitution applies wherever correlation is used inside the operational runtime civilization.

It covers:

- pattern discovery
- event association
- relationship analysis
- investigation support
- confidence reporting
- cross-runtime correlation support

It does not define implementation mechanics.
It defines the constitutional boundary that implementation must obey.

# 9 Constitutional Summary

Correlation Runtime exists to discover relationships without claiming truth.

It may:

- associate events
- detect patterns
- measure co-occurrence
- identify relationships
- suggest investigation
- produce confidence scores

It may not:

- declare causality
- approve decisions
- execute actions
- replace human analysis
- rewrite truth
- override runtime ownership

The constitution protects:

- Human Judgment
- Runtime Independence
- Auditability
- Explainability
- Uncertainty Disclosure
