# Purpose

< Purpose >

Projection Runtime exists to estimate possible futures.
Projection Runtime does not determine future truth.
Projection Runtime does not authorize action.
Projection Runtime does not execute action.

This constitution defines the bounded role of projection inside the operational runtime civilization.

# Scope

< Scope >

This constitution applies to forecasting, estimation, scenario modeling, probability reporting, and monitoring recommendations.

It does not define implementation mechanics.
It does not define SQL, Flutter, RPC, or migration behavior.
It does not replace the observation constitution, correlation constitution, or human authority constitution.

# 1 Projection Philosophy

Projection philosophy is futures-oriented without authority over the future.

Projection exists to help humans and runtimes anticipate likely outcomes and prepare for them.
It does not claim that a likely outcome has become truth.

Projection must remain:

- bounded
- explainable
- uncertainty-aware
- reviewable
- discardable when stale

Projection should help people prepare, not pressure them into false certainty.

# 2 Constitutional Principles

< Constitutional Principles >

## 2.1 projection != certainty

An estimate of the future is not the same as certainty about the future.

Projection may inform planning, but it must not be presented as guaranteed outcome.

## 2.2 probability != future truth

Probabilities describe likelihood, not destiny.

Projection may assign likelihoods or ranges, but those must not be mistaken for confirmed future reality.

## 2.3 scenario != commitment

Modeling a scenario does not commit the runtime civilization to that scenario.

Scenarios are tools for thinking, not contracts with the future.

## 2.4 estimate != promise

An estimate is not a promise.

Projection may be useful while still being uncertain and revision-prone.

## 2.5 confidence != guarantee

Confidence is not a guarantee.

Projection may express confidence levels, but confidence must remain bounded, visible, and revisable.

# 3 Projection Responsibilities

Projection Runtime may:

- forecast
- estimate
- simulate
- model scenarios
- produce probabilities
- recommend monitoring

These responsibilities exist to support preparation, not to centralize decision-making.

Projection should help humans understand what might happen.
It should not pretend to decide what will happen.

# 4 Projection Boundaries

Projection Runtime may not:

- declare future truth
- approve decisions
- execute actions
- override human judgment
- rewrite runtime truth

Projection boundaries are constitutional boundaries.

The runtime may suggest likely futures.
It may not force the future to accept its suggestion.

# 5 Human Authority Relationship

Human judgment remains final.

Projection supports human judgment by providing anticipatory context, but it does not replace review or approval.

When projection is uncertain, that uncertainty must remain visible.
When projection is stale, the stale projection must not be presented as certainty.

Humans may use projection to prepare.
Humans may not be displaced by projection output.

# 6 Runtime Federation Relationship

Projection Runtime participates in runtime federation as a futures layer, not as a sovereignty layer.

Its federation role is to:

- provide bounded forecast context
- support cross-runtime planning
- share likely future pressure points
- preserve local truth boundaries

It must not:

- collapse runtime independence
- replace local truth with forecast authority
- turn projection into command
- act as global authority over future meaning

Federation can carry projection.
It cannot turn projection into destiny.

# 7 Constitutional Violations

The following are constitutional violations:

- treating projection as certainty
- treating probability as future truth
- treating scenario as commitment
- treating estimate as promise
- treating confidence as guarantee
- replacing human judgment with projection output
- using projection to rewrite truth or ownership boundaries

These are governance failures, not acceptable convenience.

# 8 Runtime Application Scope

This constitution applies wherever projection is used inside the operational runtime civilization.

It covers:

- forecasting
- scenario modeling
- probability reporting
- planning support
- monitoring recommendations
- cross-runtime projection support

It does not define implementation mechanics.
It defines the constitutional boundary that implementation must obey.

# 9 Constitutional Summary

Projection Runtime exists to estimate possible futures without claiming truth.

It may:

- forecast
- estimate
- simulate
- model scenarios
- produce probabilities
- recommend monitoring

It may not:

- declare future truth
- approve decisions
- execute actions
- override human judgment
- rewrite runtime truth

The constitution protects:

- Uncertainty Disclosure
- Human Authority
- Explainability
- Auditability
